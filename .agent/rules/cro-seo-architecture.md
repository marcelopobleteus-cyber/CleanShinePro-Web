# CRO & SEO Architecture Rules

## Operational Restrictions & Strategic Directives

### 1. Visual & Brand Fidelity
- **Strict Adherence:** The aesthetic design (colors, typography, layout structure) must strictly replicate the *current* Clean Shine Pro design as provided in reference images.
- **Goal:** Maintain brand continuity while upgrading the underlying code architecture for performance and SEO.

### 2. SEO & Semantic Architecture
- **Hyper-Local Focus:** Structure content to target Miami, FL and specific service areas.
- **Semantic HTML5:** Use proper semantic tags (`<header>`, `<nav>`, `<main>`, `<article>`, `<section>`, `<footer>`) for all content.
- **Metadata:** Every page must have unique, optimized `<title>` and `<meta name="description">` tags.
- **Schema.org:** Implement JSON-LD structured data for LocalBusiness and Service on relevant pages.

### 3. Conversion Rate Optimization (CRO)
- **Clear CTAs:** "Book Online Now" (Primary) and "Call for a Free Estimate" (Secondary) must be prominent above the fold.
- **Trust Signals:** Display "Fully Insured & Bonded", "Eco-Friendly Products", and "24-Hour Clean Warranty" prominently.
- **Incentives:** Include the "Get $30 OFF Your First Cleaning Service!" announcement bar.

### 4. Technical Performance
- **Zero-Dependency Core:** The booking form and main interactions must use vanilla JavaScript where possible to reduce bloat.
- **Mobile-First:** All layouts must be fully responsive and optimized for mobile conversion.
- **Fast Loading:** Optimize images and code for minimal load times (Core Web Vitals focus).

### 5. Deployment & Hand-off
- **Static Output:** The final deliverable consists of static HTML/CSS/JS files ready for deployment on Netlify/Cloudflare Pages.
- **Documentation:** A comprehensive README.md must detail deployment steps.
